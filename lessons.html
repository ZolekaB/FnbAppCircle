Lesson 1: Introduction to HTML

What is HTML?

HTML is like the skeleton of a webpage. Just like your bones give your body shape, HTML gives a webpage its structure.

HTML stands for Hyper Text Markup Language.

Fun Fact: Every website you visit is made with HTML!

HTML5

HTML5 is the newest version of HTML. It's like getting the newest version of your favorite game - it has more features!

Why is HTML5 great?

  1. Works with videos and sounds
  2. Easy for people to read and computers to understand
  3. Works well on phones and tablets
  4. Can make apps that work on different devices

Getting Started

To write HTML, you need a special program called VS Code (like a notebook for writing code).

Download VS Code: https://code.visualstudio.com/

Note: VS Code works on Windows and Mac computers, but not on phones or tablets.

Thinking Like a Developer

What is Development?

Imagine you're building a Lego house. Development is similar:

  • You start with a plan (what the house should look like)
  • You break it into small pieces (walls, roof, doors)
  • You build each piece step by step
  • You check if everything fits together

Software development is the same - we build digital things piece by piece!

How Developers Think

Developers solve problems in a special way:

1. Understand the Problem

Example: "I need a website that shows my art portfolio"

2. Break It Down

- A homepage with my name
- A gallery page for my art
- A contact page

3. Solve Small Pieces

- First make the homepage
- Then make the gallery
- Then make the contact page

4. Put It All Together

Connect the pages and test everything works

Developer Mindset Tips:

  • There's always more than one way to solve a problem
  • Start simple, then add complexity
  • Expect to make mistakes - they help you learn!
  • Google is your friend - all developers search for help

What You'll Learn in This Course

  1. HTML - The building blocks of websites (like Lego bricks)
  2. CSS - Making your websites look beautiful (like paint and decorations)
  3. Development Skills - How to plan and build like a pro

Pseudocode - Thinking Before Coding

Pseudocode is like writing a recipe in plain English before you cook. It helps you plan your code without worrying about the exact programming words.

Example: Making a Sandwich

Real instructions vs. pseudocode:

Normal Instructions

  1. Take two slices of bread
  2. Spread butter on one slice
  3. Add cheese on top of the butter
  4. Put the other slice on top

Pseudocode

BEGIN MakeSandwich
    GET breadSlice1, breadSlice2
    SPREAD butter ON breadSlice1
    PUT cheese ON breadSlice1
    PUT breadSlice2 ON TOP
END MakeSandwich

Why Use Pseudocode?

  • Helps you think through the steps
  • Easier to understand than real code
  • Can be turned into any programming language
  • Great for planning with teammates

Try It Yourself:

Write pseudocode for making a cup of tea:

BEGIN MakeTea
    [Your steps here]
END MakeTea

Algorithms - Step-by-Step Solutions

An algorithm is like a recipe for solving a problem. It's a clear set of instructions to get from the problem to the solution.

Simple Algorithm Example

Problem: Determine if a number is even or odd

Algorithm Steps:

  1. Get a number from the user
  2. Divide the number by 2
  3. If there's no remainder, it's even
  4. If there is a remainder, it's odd
  5. Tell the user the result

Algorithm Characteristics

  • Clear: Each step is easy to understand
  • Finite: It has a clear ending
  • Effective: It actually solves the problem
  • General: Works for different inputs

Everyday Algorithms

You use algorithms every day without realizing:

  • Following a recipe
  • Getting dressed (underwear before pants!)
  • Brushing your teeth

Practice: Morning Routine Algorithm

Write the steps for your morning routine as an algorithm:

BEGIN MorningRoutine
    1. Wake up
    2. [Your next step]
    ...
END MorningRoutine

Software Development Life Cycle (SDLC)

Building software is like building a house. You need a plan, builders, and checks to make sure everything works.

The 5 Main Steps of SDLC

1. Planning

- What are we building?
- Who is it for?
- What features do we need?

2. Design

- How will it look?
- How will users interact with it?
- What technologies will we use?

3. Development

- Writing the actual code
- Building each feature
- Creating tests

4. Testing

- Does everything work?
- Are there any bugs?
- Is it easy to use?

5. Deployment & Maintenance

- Release to users
- Fix any issues that come up
- Add new features over time

Why SDLC Matters

  • Helps avoid mistakes and wasted time
  • Makes sure we build what users actually need
  • Helps teams work together effectively
  • Results in higher quality software

Your First Project SDLC

When you build your first website:

  1. Plan: Sketch what pages you need
  2. Design: Draw how each page should look
  3. Develop: Write the HTML/CSS code
  4. Test: Click all links, try on phone/computer
  5. Deploy: Upload to GitHub or a web host

Lesson 2: Setting up a Workspace

Your workspace is like your art table where you'll build your website. You need:

  1. A text editor (VS Code - like your pencil)
  2. A web browser (Chrome or Firefox - like your canvas)
  3. A folder to keep all your files (like a drawer for your art supplies)

Your First HTML Code

Every HTML page starts with this basic structure:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
</body>
</html>

Remember: HTML uses tags (words in <brackets>) to build web pages.

Lesson 3: Creating Paragraphs

Paragraphs are like the sentences of your webpage. They're made with the <p> tag.

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <p>Hello everyone</p>
    <p>This is fun</p> 
</body>
</html>

Try it yourself: Copy this code into VS Code and save it as "my-page.html". Then open it in a web browser!

What's happening?

  • <p> means "start a paragraph"
  • </p> means "end the paragraph"
  • Everything between them is your paragraph text

Lesson 4: Creating Headings

Headings are like titles in your webpage. HTML has 6 levels of headings, from <h1> (biggest) to <h6> (smallest).

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <h1>Home page</h1>
    <h2>This is heading 2</h2>
    <h3>This is heading 3</h3>
    <h4>This is heading 4</h4>
    <h5>This is heading 5</h5>
    <h6>This is heading 6</h6>
</body>
</html>

Pro Tip: Use <h1> for your main title, <h2> for section titles, and so on.

Why headings matter:

  • They help organize your page
  • They make your page easier to read
  • Search engines use them to understand your content

Lesson 5: Web File Formats

Just like you have different types of files on your computer (.doc for documents, .mp3 for music), websites use different file types too.

Common Web File Types:

  • .html or .htm - Web pages (like the ones you're making!)
  • .css - Files that make your page look pretty (we'll learn these later)
  • .jpg, .png, .gif - Different types of pictures

Remember: Always save your web pages with .html at the end (like "my-page.html")

Image Types:

  • .jpg - Best for photos
  • .png - Best for graphics with transparent backgrounds
  • .gif - Best for simple animations

Lesson 7: Adding an Image

Now let's add pictures to our webpage! We use the <img> tag.

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <img src="images/boat.jpg" />
</body>
</html>

Important parts:

  • <img> - This means "put an image here"
  • src="images/boat.jpg" - This tells the browser where to find the picture

Warning: Image names are case-sensitive! "Boat.jpg" is different from "boat.jpg"

Lesson 8: Resizing an Image

Sometimes pictures are too big or too small. We can change their size!

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <p>Setting image width and height</p>
    <img src="images/boat.jpg" width="150"/>
    <img src="images/boat.jpg" height="100"/>
</body>
</html>

How it works:

  • width="150" - Makes the image 150 pixels wide
  • height="100" - Makes the image 100 pixels tall
  • You can use just width or height (the other will adjust automatically)

Best Practice: Usually only set width OR height (not both) so your picture doesn't look squished.

Lesson 9: Adding More Images

You can add as many pictures to your page as you want! Just add more <img> tags.

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <h1>My Photo Gallery</h1>
    <img src="images/cablecar.jpg" height="150"/>
    <img src="images/city.jpg" height="150"/>
    <img src="images/plane.jpg" height="150"/>
</body>
</html>

Tips for multiple images:

  • Give them similar heights to make them line up nicely
  • You can add text between images with <p> tags
  • Remember to keep all images in your "images" folder

Lesson 10: Your First Hyperlink

Links let you click to go to another page or website. They're made with the <a> tag (the "a" stands for "anchor").

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <a href="http://www.google.com">Click here to go to Google</a>
</body>
</html>

Important parts:

  • <a> - This means "this is a link"
  • href="..." - This tells where the link goes
  • The text between the tags is what people see and click on

Remember: Always include https:// when linking to other websites.

Lesson 11: Adding a Second Page

Most websites have more than one page. Let's make an "About" page!

Steps:

  1. Create a new file in VS Code
  2. Save it as "about.html"
  3. Add some content

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <h2>About us</h2>
    <p>Hello and welcome to our website!</p>
</body>
</html>

Naming Tip: Your main page should always be called "index.html" - this is the page that opens automatically when someone visits your website.

Lesson 12: Linking to the Second Page

Now let's connect our two pages so we can click between them!

In index.html:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <h1>Welcome to My Site</h1>
    <a href="about.html">Click here to learn about us</a>
</body>
</html>

In about.html:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <h2>About us</h2>
    <p>Hello and welcome!</p>
    <a href="index.html">Back to Home</a>
</body>
</html>

Important: The file name in href must exactly match your other file's name (including capitalization).

Lesson 13: A Page With an Image

Let's make a special page that shows a big picture, like in a photo gallery.

Steps:

  1. Create a new file called "boat.html"
  2. Add an image tag with your picture
  3. Add a link back to the home page

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <img src="images/boat.jpg">
    <br/>
    <a href="index.html">Home</a>
</body>
</html>

New Tag: <br/> makes a line break (like pressing Enter in a document).

Lesson 14: Image Links

Now let's make pictures that you can click on! This is how photo galleries work.

Example:

<!DOCTYPE html>
<html>
<head>
</head>
<body>
    <p>Click the boat picture below:</p>
    <a href="boat.html">
        <img src="images/boat.jpg" height="150"/>
    </a>
</body>
</html>

How it works:

  • Put the <img> tag inside the <a> tags
  • When you click the image, it will go to the page in the href
  • You can use this to make thumbnail galleries
